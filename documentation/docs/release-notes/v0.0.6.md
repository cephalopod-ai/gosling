# Gosling v0.0.6

Gosling v0.0.6 is a hardening release for the desktop app, local backend lifecycle, session recovery, code execution configuration, MCP app proxying, and security defaults.

## Highlights

- Desktop now records launched backend processes and cleans up stale desktop `gosling serve --platform desktop` processes at startup.
- Desktop quit now runs an awaited closeout path for backend leases, recorded backend processes, power blockers, and shortcuts.
- macOS now uses the same single-instance lock behavior as other platforms; second launches focus the existing window or route protocol URLs.
- Packaged desktop builds now allow loopback HTTP and WebSocket connections in the static Content Security Policy so the renderer can reach the local ACP backend.
- Local summarization gained a worker path, durable file-backed facts, per-backend routing, and compacted session resume paging.
- Code execution runtime configuration is available through CLI and desktop settings, including V8 runtime selection.
- MCP app proxy and ACP surfaces gained route, custom request, SDK, OpenAPI, and renderer updates.
- Goose compatibility support was expanded for documentation skills and catalog normalization.

## Fixes And Hardening

- Tool inspection now fails closed by requiring approval when an inspector errors.
- Default agent permissions were tightened.
- Plugin installation now guards `git clone` with `--` to prevent option injection.
- Google retry delays are clamped and provider auth HTTP clients are bounded.
- Secret writes are atomic.
- Desktop startup diagnostics and troubleshooting docs now use the `gosling-serve-startup-*.json` diagnostic file naming.
